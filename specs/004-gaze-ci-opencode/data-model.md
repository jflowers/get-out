# Data Model: Add Gaze Quality Analysis to GitHub CI via OpenCode and Zen

**Feature**: 004-gaze-ci-opencode  
**Date**: 2026-03-12

## Summary

This feature introduces no new persistent data entities. It is implemented entirely as a change to the CI workflow configuration file (`.github/workflows/ci.yml`). There are no new database tables, files, structs, or storage schemas.

## Configuration Values (Non-Persistent)

The following values exist only within the workflow YAML as configuration inputs. They are not stored, read from a database, or persisted beyond the CI run.

| Value | Type | Location | Description |
|---|---|---|---|
| `OPENCODE_API_KEY` | Secret string | GitHub Actions secret store | Zen authentication key for the OpenCode CLI. Set via repository Settings → Secrets → Actions. Never appears in source code or logs. |
| `coverage.out` | File (ephemeral) | GitHub Actions workspace root | Go coverage profile generated by the `Test` step. Consumed by the `Gaze quality report` step in the same job. Discarded when the runner cleans up. |
| `--max-crapload` | Integer | Workflow YAML | Hard gate: maximum number of functions whose CRAP score exceeds the CRAP threshold. Initial value: 10. |
| `--max-gaze-crapload` | Integer | Workflow YAML | Hard gate: maximum number of functions whose GazeCRAP score exceeds the GazeCRAP threshold. Initial value: 5. |
| `--min-contract-coverage` | Integer (percent) | Workflow YAML | Hard gate: minimum average contract coverage across all analyzed functions. Initial value: 50. |
| `--model` | String | Workflow YAML | OpenCode Zen model identifier passed to `gaze report`. Value: `opencode/claude-sonnet-4-6`. |

## Ephemeral Artifacts (CI Run Lifetime Only)

| Artifact | Produced By | Consumed By | Lifetime |
|---|---|---|---|
| `coverage.out` | `Test` step (`go test -coverprofile`) | `Gaze quality report` step (`--coverprofile`) | Single CI job execution |
| Temp dir `gaze-opencode-*` | `gaze report` adapter internally | `opencode run --dir` subprocess | Cleaned up by `defer os.RemoveAll` within `gaze report` |
| GitHub Step Summary entry | `gaze report` (writes via `GITHUB_STEP_SUMMARY` env var) | Developer reads in GitHub Actions UI | Retained by GitHub per their standard retention policy |

## State Transitions

None. This feature introduces no stateful processes, no checkpoints, and no resume logic.
