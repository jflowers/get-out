# Data Model: Distribution, Config Migration & Self-Service Commands

**Feature**: 003-distribution  
**Date**: 2026-03-11

This document describes the entities, file schemas, and state transitions introduced or modified by this feature. All state is stored as JSON files on disk in `~/.get-out/` (or the directory specified by `--config`).

---

## Entities

### 1. Settings (`settings.json`)

Stores non-secret user preferences. Optional — all fields have defaults.

**Schema** (after this feature):

```json
{
  "folder_id": "string",
  "google_credentials_file": "string"
}
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `folder_id` | string | No | `""` | Google Drive folder ID where exports are written. When non-empty, `export` uses this as the default destination without requiring `--folder-id`. |
| `google_credentials_file` | string | No | `<config_dir>/credentials.json` | Override path to the OAuth credentials JSON file. |

**Validation rules**:
- `folder_id`, if non-empty, MUST match `^[a-zA-Z0-9_-]{28,}$`
- `google_credentials_file`, if non-empty, MUST be an absolute path

**Go type** (in `pkg/config/types.go`, `Settings` struct — adds `FolderID` field):
```go
type Settings struct {
    GoogleCredentialsFile string `json:"google_credentials_file,omitempty"`
    FolderID              string `json:"folder_id,omitempty"`   // NEW
}
```

**File permissions**: written with process umask default (typically `0644`).

### 2. Config Directory (`~/.get-out/`)

The root container for all user-specific state. Not a JSON file — a filesystem directory.

| Attribute | Value |
|-----------|-------|
| Default path | `~/.get-out/` |
| Legacy path | `~/.config/get-out/` (migration source only) |
| Permissions | `0700` (owner rwx only) |
| Created by | `get-out init` |

**Contents**:

| File | Created by | Permissions | Required |
|------|-----------|-------------|----------|
| `settings.json` | `init` (template) | umask default | No (defaults apply) |
| `conversations.json` | `init` (template) | umask default | Yes (for `list`, `export`) |
| `people.json` | `discover` | umask default | No |
| `credentials.json` | user (manual copy) | **0600** | Yes (for `auth login`) |
| `token.json` | `auth login` | **0600** | Yes (for `export`, `doctor` Drive check) |
| `export-index.json` | `export` | umask default | No (created on first export) |

### 3. SecretStore (in-memory abstraction)

Not persisted directly — controls *how* secrets are persisted.

| Attribute | Value |
|-----------|-------|
| Interface | `pkg/secrets.SecretStore` |
| Implementations | `KeychainStore` (OS keychain), `FileStore` (0600 files) |
| Selection | `NewStore(noKeyring bool)` — probes keychain on startup; falls back to `FileStore` |
| Service name (keychain) | `com.jflowers.get-out` |
| Probe key | `__get_out_probe__` (write/read/delete cycle at startup) |

**Well-known keys**:

| Key constant | `KeychainStore` storage | `FileStore` storage |
|-------------|------------------------|---------------------|
| `KeyOAuthToken = "oauth-token"` | macOS Keychain item | `token.json` (0600) |
| `KeyClientCredentials = "credentials-json"` | macOS Keychain item | `credentials.json` (0600) |

**Backend selection**:

```
noKeyring flag set?
    YES → FileStore (unconditional)
    NO  → probe keychain (write/read/delete __get_out_probe__)
            probe OK  → KeychainStore
            probe fail → FileStore (silent fallback)
```

**Go type** (in `pkg/secrets/store.go`):
```go
type SecretStore interface {
    Get(key string) (string, error)   // ErrNotFound if absent
    Set(key, value string) error
    Delete(key string) error          // no error if absent
}
```

### 4. OAuth Credentials (`credentials.json` / keychain key `credentials-json`)

Google Cloud OAuth 2.0 "Desktop app" client credentials. Obtained manually from Google Cloud Console; not generated by get-out.

**Schema**: Standard Google `installed` client credential format (not modified by this feature).

```json
{
  "installed": {
    "client_id": "string",
    "client_secret": "string",
    "redirect_uris": ["string"],
    "auth_uri": "string",
    "token_uri": "string"
  }
}
```

**Storage**: User places `credentials.json` in `~/.get-out/`. On `get-out init`, if `KeychainStore` is active, `secrets.Migrate()` reads the file, writes it to the keychain under key `credentials-json`, and (after interactive confirmation or non-interactive notice) removes the file from disk. When `FileStore` is active, the file remains on disk with mode `0600`. `auth login` reads credentials via `store.Get(KeyClientCredentials)`.

### 5. OAuth Token (`token.json` / keychain key `oauth-token`)

Google OAuth 2.0 access + refresh token. Written by `get-out auth login`; silently refreshed by `get-out auth status` and `get-out export`.

**Schema**: Standard `golang.org/x/oauth2` token format (not modified by this feature).

```json
{
  "access_token": "string",
  "token_type": "Bearer",
  "refresh_token": "string",
  "expiry": "RFC3339 datetime"
}
```

**State transitions**:

```
[absent]
    │  get-out auth login (success)
    ▼
[valid]
    │  time passes, access_token expires
    ▼
[expired-refreshable]    ←── refresh_token still present
    │  get-out auth status / export (silent refresh)
    ▼
[valid]
    │  refresh_token revoked or expires
    ▼
[expired-terminal]       ←── must re-run get-out auth login
```

**Storage**: Written by `auth login` via `store.Set(KeyOAuthToken, ...)`. When `FileStore` is active, written to `token.json` with mode `0600`. When `KeychainStore` is active, stored in OS keychain (no file on disk).

### 6. Doctor Check Result (in-memory only)

Not persisted. Represents the outcome of one `doctor` health check.

| Field | Type | Values |
|-------|------|--------|
| `name` | string | Human-readable check description |
| `status` | enum | `pass`, `warn`, `fail` |
| `message` | string | Detail shown on the check row |
| `fix` | string | Corrective action; empty if status is `pass` |
| `path` | string | Resolved file/resource path; shown in `--verbose` mode |

**Aggregate behaviour**:
- `failCount > 0` → process exits with code 1
- `failCount == 0` → process exits with code 0 (even if `warnCount > 0`)

## State Transitions: Init Flow

```
User runs: get-out init
    │
    ├─► ~/.get-out/ does not exist?
    │       YES → mkdir -p ~/.get-out/ (mode 0700)
    │       NO  → directory already exists, continue
    │
    ├─► ~/.config/get-out/ exists?
    │       YES → for each managed file in old dir:
    │               if file NOT in ~/.get-out/ → copy (credentials.json/token.json with 0600)
    │               if file already in ~/.get-out/ → skip (never overwrite)
    │             print migration notice
    │       NO  → skip migration
    │
    ├─► conversations.json absent in ~/.get-out/?
    │       YES → write template {"conversations":[]}
    │
    ├─► settings.json absent in ~/.get-out/?
    │       YES → write template {}
    │
    ├─► --non-interactive NOT set AND settings.folder_id == ""?
    │       YES → prompt user for folder ID (huh form with validation)
    │             write folder_id to settings.json
    │       NO  → skip prompt
    │
    ├─► secrets.Migrate(store, configDir, interactive, ...)
    │       → if token.json on disk + not in store:
    │             store.Set(KeyOAuthToken, contents) → os.Remove(token.json)
    │       → if credentials.json on disk + not in store:
    │             store.Set(KeyClientCredentials, contents)
    │             if interactive → huh.Confirm("Delete credentials.json?")
    │             if non-interactive → print notice ("manually delete when ready")
    │       (idempotent — safe to run every init)
    │
    └─► print "Next Steps" summary
```
